# Design System — Gander Studio
> Version: 1.0.0 | Updated: 2026-04-02 | Generated by: generate-design

## Constitution

Non-negotiable rules for this app's visual identity.

- **Dark mode only:** never use white or light-gray backgrounds — minimum surface is `#0f1117`
- **No CRT scanlines:** the `body::before` scanline overlay is prohibited in Studio Clarity; remove before any sprint ships
- **No glow shadows:** `box-shadow` glow effects (e.g. `0 0 12px rgba(...)`) are prohibited; use `1px solid` border-on-hover instead
- **No serif fonts:** Optima and Palatino are deprecated; all text uses `Geist Variable` at varying weights
- **Icons from lucide-react only:** no inline SVG, no other icon libraries
- **Minimum touch target: 44×44px** for all interactive elements
- **Role colors are preserved:** the materia color system (green/yellow/blue/purple/red/orange) is functionally meaningful and must not be changed without a design decision record

## Component Rules

| Component | Rule |
|-----------|------|
| Primary button | Shadcn `<Button variant="default">` — uses `--color-primary` |
| Destructive action | Shadcn `<Button variant="destructive">`, always paired with a confirmation step |
| Text input | Shadcn `<Input>` + `<Label>` — never a raw `<input>`; border becomes `--color-border-active` on focus |
| Popover / dropdown | Shadcn `<Popover>` — background `--color-surface-elevated`, `1px solid --color-border`, radius `--radius-md`; no glow shadow |
| Dialog / modal | Shadcn `<Dialog>` — overlay `rgba(0,0,0,0.6)`, panel `--color-surface-elevated` |
| Card | `background: --color-surface`, `border: 1px solid --color-border`, radius `--radius-md`; on hover: border becomes `1px solid --color-border-active` — no box-shadow glow |
| Empty state | Icon (lucide) + heading (`lg`) + body text (`sm`) + CTA — all four required |
| Loading state | Shadcn `<Skeleton>` shimmer for content areas; no spinner for page-level loads |
| Error state | Shadcn `<Alert variant="destructive">` with a descriptive message; left-border accent `--color-error` |
| Role / type tag | Pill-shaped `<span>` with materia color as `color` and `background: rgba(color, 0.12)`, `border: 1px solid rgba(color, 0.25)` — no solid fill |
| Agent code badge | Monospace `2-char` code, `--color-surface-elevated` background, `--color-text-muted` text |
| Nav item (sidebar) | Left `2px solid` border in `--color-primary` when active; background `rgba(74,143,168,0.10)`; no border when inactive |
| Collapsible sidebar | Open: `240px`; Collapsed: `56px` (icons only); animated via CSS `width` transition `200ms ease` |
| Page title | `lg` size, weight `600`, color `--color-text`; preceded by `2px × 18px` vertical rule in `--color-primary` |
| Section label | `xs` size, `letter-spacing: 0.12em`, `text-transform: uppercase`, color `--color-text-faint` |

## Design Integrity Notes

- **Migration from FF7 tokens:** existing `--void`, `--sf`, `--mt`, `--dg`, etc. in `globals.css` are the legacy system. Studio Clarity replaces them with semantic `--color-*` tokens. During migration, both systems may coexist in `globals.css` — legacy tokens should be removed file-by-file as components are updated.
- **No raw hex in `.ts`/`.tsx`:** all color values must reference a CSS variable. Constants files (`browse.ts`, `compose.ts`, `edit.ts`) that currently contain `rgba(84,153,181,...)` literals must be updated to `rgba(var(--color-primary-rgb), 0.N)` or extracted to semantic token names.
- **Geist Variable is already installed** (`@fontsource-variable/geist`) — no new dependency needed. Update `globals.css` to set `font-family: "Geist Variable", ...` on `:root` and remove Optima/Segoe UI declarations.

## Gaps Addressed

### G2 → Merged with G5: E2E Assertion Targeting (Tier 2 spec authoring best practices)

**Post-mortem root causes:**
- **G2** (line ~947): "QA verified tab-stub matchers corrected — initial e2e asserted `toBeVisible` on zero-dimension stubs; fixed to `toBeAttached`"
- **G5** (line ~1004–1008): "E2E tests coupled to incidental fixture state; both t5b and t6b failures traced to row selection (picked first row; fixture varied)"

**Rationale for merge:** Both gaps reflect the same underlying lesson — Playwright Tier 2 test specs must assert on user-visible primary effects (rendered DOM presence, interaction state), not side effects or incidental fixture properties. The pattern covers three specific pitfalls that have shipped past full audit gates:

1. **Zero-dimension stub matcher collapse** — `toBeVisible()` assertion on a div with `width:0; height:0` passes (box exists, has zero dimensions) but the user cannot see or interact with it. Correct matcher: `toBeAttached()` (element is in DOM) or `toBeFocused()` (for focus targets). Applied to: tab stubs in t3b SessionsRouter, t5a detail-shell, t5b Overview/Table tabs.

2. **First-row fixture coupling** — Test selects `page.locator('tbody tr').first()` assuming it has specific properties (agents > 0 for table assertion). When fixture varies, assertion fails on incidental property, not feature behavior. Correct pattern: select by semantic role (`role=button`) or aria-label, document assumptions explicitly in test comment (e.g., "row must have agents > 0 for TableTab assertions to run").

3. **Rendered-but-invisible UI-primitive token-system collision** — Shadcn primitives inherit `:root` light-theme tokens (text-foreground ≈ black) when not explicitly wrapped in dark-theme class/attribute. Component renders (visible in DOM inspection, passes e2e assertOnElement), but text color collapses into surface color (t6b editor textarea). Correct pattern: explicit inline verbatim token override with code comment showing computed color:
   ```tsx
   {/* FF7 dark surface --sfm (teal), require --w (light) text. Contrast ≥ 4.5:1 WCAG AA */}
   <textarea style={{ color: 'var(--w)', background: 'var(--sfm)', ... }} />
   ```
   Include a `getComputedStyle` readability snippet in e2e guard (`assert(getComputedStyle(elem).color !== getComputedStyle(elem).backgroundColor)`).

**Fix applied to frontend.md §E2E Assertion Targeting:**

Added new Tier 2 spec authoring section documenting all three pitfalls with:
- Bad/good code examples for each
- Playwright matcher choice matrix (toBeVisible vs toBeAttached vs toBeFocused)
- Fixture-independence checklist (semantic role-based selection, documented assumptions)
- Token-system collision detection snippet (getComputedStyle guard for Shadcn consumers)

This ensures FE agents self-catch these patterns before audit, reducing post-delivery regressions that escape audit gates.

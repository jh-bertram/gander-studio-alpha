## G2 Root-Cause Diagnosis (summary)

The hook reads `max(seq)` over the SINGLE UTC-dated `events_file` (line 145 — `read_text()`, not a
glob), so cross-file contamination is ruled out. It writes `last_seq + 1`, contiguous by
construction. `33 + 1 = 34`, never 99 — the hook has no arithmetic path to 99. Therefore `seq:99`
was written by an external actor. The only external writer of COMPLETE events to this log is the
backend agent self-logging (G1, which the pre-patch backend.md instructed). On the next hook
invocation the hook sees `max(seq)=99`, the task is already terminal, and it dedup-skips (line 161).
**The 33→99 jump is entirely a downstream symptom of G1; the hook is correct.** Hence: fix the cause
in backend.md (G1) and add only a visibility warning to the hook.
